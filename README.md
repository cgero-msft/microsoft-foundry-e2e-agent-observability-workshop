# Build _Trustworthy_ Multi-Agent Apps On Microsoft Foundry 

This repository provides the source code and content required for breakout talks and workshops that showcase the Microsoft Foundry observability story with a real-world use case.

## Workshop Outline

In this hands-on workshop, you'll build the observability muscle that separates demo-friendly agent prototypes from production-ready agent deployments. 
We'll work through four capabilities every agent builder needs to have in their arsenal:

|  | AI Engineer Europe |
|:---|:---|
|Tracing  |Step through your agent's reasoning, see every tool call and decision, and pinpoint exactly where things went wrong. |
|Evaluations |Move beyond "looks good to me". Measure task adherence, groundedness, and safety with automated scoring — interactively (in dev) and continuously (in production) |
|Red Teaming |Attack your own agents before someone else does. Run automated adversarial scans to test jailbreaks, prompt injection, and content safety risks – then mitigate them with the right guardrails. |
|Fleet Monitoring | Go from watching a single agent to managing an entire fleet . Think of having centralized dashboards, compliance guardrails, cost tracking, and anomaly detection across your entire agent inventory.|
| | |

You’ll leave the workshop with a better mental model for agent observability, and hands-on experience with relevant tools and workflows to observe, safeguard, and evolve, your next agentic AI solution.
